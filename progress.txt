## How to work here
- pnpm workspace. From repo root: `pnpm install`.
- Contracts: `cd packages/contracts && npx hardhat compile && pnpm test`.
  Single file: `npx hardhat test test/GameBase.test.ts` (14), `test/CoinFlip.test.ts` (23).
- Tests use Hardhat's in-process network — no RPC/secrets needed. (NOT Foundry; native forge fuzz
  is a deferred follow-up per the spec's testing note.)

## Gotchas
- 943/shanghai: every DEPLOYED game contract needs evmVersion:'shanghai' in hardhat.config
  overrides — viaIR+cancun emits MCOPY which reverts on chain 943. GameBase is inherited, so it
  needs the override too.
- seed = keccak256(concat(revealed secrets)) (Random.sol cast). Off-chain reductions must match
  exactly: coinflip `seed & 1`; raffle `1 + (seed % 256)`. The Plan-2 cross-layer parity test
  guards this.
- onCast fires only when the seed fully forms; onChop only when it never forms. So a no-contest
  round (seed set, zero reveals) had no chop -> pot goes to the whole declared subset; onChop is
  the liveness/refund path only.
- 1Password vault auto-locks during long subagent runs, breaking commit signing AND `git push`
  (SSH agent). Keep it unlocked, or commit/push right after each task while the window is warm.
