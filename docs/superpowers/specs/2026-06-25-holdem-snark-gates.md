## Gate 1 ŌĆö SNARK reveal-share dispute track ŌĆö **CLOSED** Ō£ģ

A contested decryption-**SHARE** demand is now answerable on-chain, so an honest seat can
never be force-folded on the clock for a share it actually performed.

**Insight.** A decryption share's correctness *is* a ChaumŌĆōPedersen DLEQ statement:
`share d = c1┬Ęsk` and the responder's deck pubkey `pk = G┬Ęsk` share the SAME secret `sk`
(`log_G(pk) == log_{c1}(d)`). The off-chain prover already emits exactly this proof
(`zk-core/src/chaumPedersen.ts#proveShare ŌåÆ {t1, t2, z}`).

**Curve decision (important, and a correction to the original framing).** The vendored
`vendor/uzkge/verifier/ChaumPedersenDLVerifier.sol` is **EdOnBN254 (Baby JubJub)** ŌĆö the
WRONG curve for our live deck, which is **secp256k1** (`@noble/curves`). Wiring that vendored
verifier would *not* verify any real share. So Gate 1 is closed with a **curve-matched
secp256k1** verifier instead, sidestepping the curve mismatch honestly:

- `contracts/zk/lib/EllipticCurve.sol` ŌĆö minimal Jacobian secp256k1 point arithmetic
  (add/double/scalar-mul, on-curve guard; Witnet-derived, MIT).
- `contracts/zk/lib/RevealShareDLEQ.sol` ŌĆö exact on-chain mirror of `chaumPedersen.ts`:
  recomputes the FiatŌĆōShamir challenge `e = keccak(tagŌĆ¢compressed(pk,c1,c2,d,t1,t2)ŌĆ¢keccak(ctx)) mod n`
  byte-for-byte and checks `G┬Ęz == t1+pk┬Ęe` and `c1┬Ęz == t2+d┬Ęe`.
- `HoldemTableN.respondWithShare(tableId, deck, share, proof)` ŌĆö binds the passed deck to the
  contested `deckCommitment` (`_deckHash`, the compressed-wire mirror of `dealSeq.deckCommitment`),
  reads the demandSeat's registered `deckKey`, rebuilds `ctx = ctxFor(tableId, slot)`
  (`LibString`), and verifies the DLEQ. Correct share ŌåÆ dispute clears to Live; forged/incorrect
  ŌåÆ `BadShareProof` revert (clock continues to forced-fold). Seats register their Baby-JubJubŌĆ”
  *secp256k1* deck pubkey via `registerDeckKey()` while Forming (immutable across live play).

**Gas:** DLEQ verify ~1.26M; full `respondWithShare` ~2.27M (incl. 52-card commitment rehash).
Well under the ~15.6M the design rejected for the vendored EdOnBN254 CP-DL path. Cold dispute
path; an `ecrecover`-trick optimization for eq (1) (base G) is possible later but unneeded.

**Tests (all green, real crypto, no always-pass stubs):**
- `test/foundry/RevealShareDLEQ.t.sol` (8) ŌĆö parity against REAL noble vectors: challenge
  matches to the bit, honest verifies, forged-z / wrong-share / off-curve / ctx-swap all reject.
- `test/foundry/HoldemShareDispute.t.sol` (5, `--ffi`) ŌĆö true end-to-end: a real deck + share +
  DLEQ proof generated by the live off-chain prover (`scripts/gen-share-dispute.mts`) bound to
  the runtime tableId/slot; correct share resolves the dispute AND `resolveTimeout` then reverts
  (honest seat un-strandable); forged share / wrong deck / wrong seat all revert.
- `test/foundry/HoldemTableN.t.sol` ŌĆö `registerDeckKey` guards + wrong-status revert (15 total).
- `zk-core/test/revealShareOnchain.test.ts` (4) ŌĆö TS mirror pins both DLEQ equations + the
  compressed deck-commitment encoding to the prover.

Run: `forge test` (default suite, 78 pass) and
`FOUNDRY_PROFILE=ffi forge test --match-path 'test/foundry/HoldemShareDispute*' --ffi`.

**Convergence note.** If/when Gate 2 re-homes the deck to Baby JubJub, the share dispute can
switch to the vendored EdOnBN254 `ChaumPedersenDLVerifier` (then curve-correct) or the SDK's
`reveal_card_with_snark` (Groth16, ~225k gas). The secp256k1 verifier here is correct for the
deck we ship TODAY; it is replaceable behind `respondWithShare` after the re-home.

## Gate 2 ŌĆö full deck-permutation SNARK shuffle ŌĆö **SPIKE (prover found + run; re-home is the work)**

**The prover gap is closeable.** The missing off-chain Zypher uzkge prover is published:
**`@zypher-game/secret-engine`** (npm, **v0.3.0**, WASM ~4.6 MB; from `github.com/zypher-game/uzkge`).
Confirmed exported API (from the package `.d.ts`):

```
generate_key() -> { sk, pk, pkxy }            aggregate_keys(publics) -> joint
init_masked_cards(joint, num)                 mask_card(joint, index) / verify_masked_card(...)
init_prover_key(num)                          init_reveal_key()
shuffle_cards(joint, deck) -> { cards, proof }      verify_shuffled_cards(d1, d2, proof) -> bool
refresh_joint_key(joint, num) -> string[]     // == the on-chain `publicKeyCommitment` (pkc)
reveal_card(sk, card) / reveal_card_with_snark(sk, card) / verify_revealed_card / unmask_card
card_to_index / index_to_card
```

The package bundles a `Verifiers.sol` whose `ZgShuffleVerifier.verifyShuffle(bytes proof,
uint256[] inputs, uint256[] publicKeyCommitment)` is **byte-identical in signature** to the
already-vendored `vendor/uzkge/shuffle/ShuffleVerifier.sol#verifyShuffle` (and our
`ShuffleVerifier52.verify52`). `refresh_joint_key(joint, 52)` produces the 24-word `pkc` the
verifier wants (matching its 12 `CM_SHUFFLE_PUBLIC_KEY` slots).

**PoC actually run (this session, isolated):** `generate_key`, `aggregate_keys`,
`init_prover_key(52)` (~**12.9 s** one-time proving-key setup), and `init_masked_cards(joint,52)`
all succeed. `shuffle_cards` reaches the prover core; it currently panics on `MaskedCard` input
marshalling (the WASM wants the bare `MaskedCard` tuple in the SDK's field order `{e2,e1}`, not
the `{card, proof}` objects `init_masked_cards` returns) ŌĆö a solvable shape detail, not a
fundamental blocker. So: **the prover is real and runs; remaining work is integration, not
existence.**

**Two genuine blockers (the real cost):**

1. **Curve re-home (the load-bearing one).** The SDK deck crypto is **Baby JubJub / ed_on_bn254**,
   not secp256k1. Adopting real SNARK shuffle means moving the off-chain deck off secp256k1 onto
   the SDK's ElGamal ŌĆö i.e. implement a `ZypherDeckProvider` behind the EXISTING
   `MaskedDeckProvider` seam (`zk-core/src/maskedDeck.ts`), replacing `AttestedElGamalDeck`:
   `keygenŌåÆgenerate_key`, `aggregateŌåÆaggregate_keys`, `initialDeckŌåÆinit_masked_cards`,
   `shuffleŌåÆshuffle_cards`, `verifyShuffleŌåÆverify_shuffled_cards` (off-chain) or the vendored
   on-chain verifier (disputes), `shareŌåÆreveal_card[_with_snark]`, `unmaskŌåÆunmask_card`. The
   seam exists precisely for this swap; downstream (`deckN.ts`, `revealN.ts`, `dealSeq.ts`) is
   curve-agnostic over the seam. Knock-on: Gate 1's share dispute moves to the vendored EdOnBN254
   verifier (now curve-correct) ŌĆö see Convergence note above.
2. **License.** Underlying `uzkge` is **GPLv3**; the npm `@zypher-game/secret-engine` self-tags
   **MIT** (bundled `Verifiers.sol` is MIT). A GPLv3-derived WASM redistributed as MIT is a
   genuine legal ambiguity ŌĆö **must be cleared with counsel before shipping** the prover. The
   on-chain verifiers are already vendored, so on-chain verification has no new license exposure.

**Remaining-work plan (concrete):**
- P6.1 Pin `@zypher-game/secret-engine@0.3.0`; resolve the `MaskedCard` marshalling against the
  SDK's own example (`rotcan/zypher-zoo-clash`) ŌĆö get `shuffle_cards`+`verify_shuffled_cards`
  green in a vitest, in a Web Worker (WASM, off the main thread ŌĆö same posture as the PoW
  grinder per repo memory).
- P6.2 **On-chain PoC gate:** feed a freshly-generated `{proof, inputs=beforeŌĆ¢after (416),
  pkc=refresh_joint_key (24)}` into the vendored `ShuffleVerifier52.verify52` in Foundry and
  assert it verifies. This also confirms whether the v0.3.0 proving key matches the vendored
  `VerifierKey_52` (the vendored fixtures came from an older pinned wasm ŌĆö if keys drifted,
  re-vendor the matching `VerifierKey_52`/`Extra` from the package).
- P6.3 `ZypherDeckProvider implements MaskedDeckProvider` (Baby JubJub) behind the seam; run the
  whole `deckN`/`revealN`/`dealSeq` suite against it (it is curve-agnostic). Measure full-hand
  perf (shuffle proof time ├Ś N shufflers) for the tempo budget.
- P6.4 Migrate Gate 1's share dispute to the vendored EdOnBN254 `ChaumPedersenDLVerifier` /
  `reveal_card_with_snark` once the deck is Baby JubJub; retire `EllipticCurve.sol`/secp256k1
  `RevealShareDLEQ.sol` (keep until then).
- P6.5 Legal sign-off on the GPLv3ŌåÆMIT question before any mainnet/real-money ship.

**Honesty line:** no shuffle proof was faked and no verifier stubbed to pass. The vendored

## Gate 2 spike ŌĆö execution status (2026-06-25, `feat/holdem-gate2-shuffle`)

license review**, P6.5 unchanged). Every file that touches the prover carries a top-of-file GPL
provenance note; the prover is loaded lazily behind the seam so it can be swapped/removed.

- **P6.1 MaskedCard marshalling ŌĆö DONE.** The `shuffle_cards` panic had TWO causes: (1) the deck

  `shuffle_cards`** ŌĆö it loads the joint key into prover state; skipping it makes shuffle panic
